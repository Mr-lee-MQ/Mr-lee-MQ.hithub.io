CSS

发展史

  • CSS1.0
  • CSS2.0 DIV（块） + CSS，HTML 和CSS结构分离的思想，网页变得简单，SEO
  • CSS2.1 浮动，定位
  • CSS3.0 圆角，阴影，动画 浏览器兼容性

入门

  • style 存放CSS代码
    • 选择器{ 声明1；声明2……. }
    • 单独写一个CSS文件，在HTML中用link引入
<h1>
     我是标题
</h1>
<link rel="stylesheet" href="CSS/CSS1.css">
h1{
    color: aquamarine;
}

CSS的优势

  • 内容和表现分离
  • 网页结构表现统一，可以实现复用
  • 样式十分的丰富
  • 建立使用独立于html的css文件
  • 利用SEO，容易被搜索引擎收录

CSS的四种引入方式

优先级：就近原则，谁离代码近，谁的优先级越高

  • 行内样式：在元素标签中，编写一个style属性，编写样式即可
<h1 style="color: red" >我是标题</h1>
  • 内部样式：
<h1>我是标题</h1>

<style>
    h1 {
       color: pink;
    }
</style>
  • 外部样式
<h1>
     我是标题
</h1>
<link rel="stylesheet" href="CSS/CSS1.css">


h1{
    color: aquamarine;
}

拓展：外部样式两种写法

  • 链接式

选择器

作用：选择页面上的某一个或者某一类元素

基本选择器

不遵循就近原则。id选择器>类选择器>标签选择器

  • 标签选择器：选择同种类型标签，只能对同一类标签进行操作
<style>
    h1{
        color: pink;
    }
    p{
        color: aqua;
    }
</style>

<h1>
    第一行
</h1>
<h1>
    第二行
</h1>
<p>
  第三行
</p>
  • 类选择器： （.class名称{}) 为标签定义class 可以多个标签归类，是同一个class，可以复用。
<style>
    .one{
        color: aqua;
    }
    .two{
        color: pink;
    }
    .three{
        color: red;
    }
</style>
<h1 class="two">
    第二行
</h1>
<h1 class="three">
    第三行
</h1>
<p class="one">
    第一行
</p>
  • id选择器： (#id名称{}) id保证全局唯一
<style>
    #abc{
        color: red;
    }
    #bca{
        color: pink;
    }
    #cba{
        color: aqua;
    }
</style>
<h1 id="abc">第一行</h1>
<h1 id="bca">第一行</h1>
<h1 id="cba">第一行</h1>

层次选择器

  • 后代选择器：在某个元素的后面。
    • body（层次） 标签{ }
<style>
    body h1{
        color: pink;
    }
</style>
  • 子选择器：只有一代（仅下一个层次）
    • body（层次）>标签{ }
<style>
    body >h1{
        color: aqua;
    }
</style>
  • 相邻兄弟选择器：同一层次，只有一个（仅对下）
<style>
    .active +h1{
        color: red;
    }
</style>

  • .（class值） + 标签{ }

  • 通用兄弟选择器：同一层次，向下所有

<style>
    .active~ h1{
        color: red;
    }
</style>
  • .(class值) ~ 标签{ }

浮动

  • 标准文档流

块级元素：独占一行

h1~h6    p     div  列表

行内元素：不独占一行

span   a    img

行内元素可以被包含在块级元素中。

  • display

    • block： 块元素
    • inline 行内元素
    • inline-block 是块元素，但可以内联，在一行
    • none

  • 浮动 float

float: right;
float: left;
  • clear
clear:right;      右侧不允许有浮动元素
clear:left;       左侧不允许有浮动元素
clear:both;       两侧都不允许有

父级边框塌陷问题 解决方案：

  • 增加父级元素的高度
#father{
    bored:1px #000 solid;
    height:800px;
}
  • 增加一个空的div标签，清除浮动
<div class="clear"></div>

.clear{
    clear:both;
    margin:0;
    padding:0;
}
  • 在父级元素中加一个 overflow（自动溢出）
    • hidden 隐藏
  • 父类增加一个伪类：after
#father:after{
    content:'';
    display:block;
    clear:both;
}

小结：

  • 浮动元素后面加空div

简单，代码中尽量避免空div

  • 设置父级元素的高度

简单，元素假设有了固定的高度，就会被限制

  • overflow

简单，下拉的一些场景避免使用

  • 父级添加一个伪类：after（推荐）

写法稍微复杂，但是没有缺点

对比

  • display

方向不可控制

  • float

浮动起来会脱离标准文档流

绝对定位

position:absolute

  • 定位：基于xxx定位，上下左右
    • 没有父级元素定位的前提下，相对于浏览器定位
    • 假设父级元素存在定位，我们通常会相对于父级元素进行偏移
    • 在父级元素范围内移动
  • 相对于父级或浏览器的位置，进行指定的偏移，绝对定位的话，他不在标准文档流中，原来的位置也不会被保留。

固定定位

position:fixed;

元素的位置相对于浏览器窗口是固定位置。即使窗口是滚动的它也不会移动。fixed表示脱离了正常的文档流。
